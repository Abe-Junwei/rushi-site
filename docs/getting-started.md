# 使用说明

这份指南带你从安装走到第一次交付导出。Rushi 把录音、转写稿、术语表、纠错记忆和交付文件放在同一个本地项目里，适合课程录音、访谈、口述整理，以及需要反复听写校对的长音频。

默认路线是本机转写：音频与识别都在你的电脑上完成。需要厂商能力时，再按 Provider 单独配置在线 STT。

## 1. 安装与首次启动

当前正式版为 **v1.0.0**。先按平台安装，再在首次启动向导中完成模型 seed；准备完成后即可本机转写。

### Windows 安装步骤

1. 下载 **离线安装包 zip**（约 1.4 GB）。
2. 右键 → **全部提取**（或解压到任意空文件夹），得到类似结构：
   - `如是我闻_1.0.0_Windows_x64_安装包.exe`
   - `resources/bundled-asr-models/…`
3. 双击运行 **同级** 的 `*_安装包.exe`，按向导完成安装。
4. 首次启动后按向导完成模型 seed；即可本机转写。

可选：有 NVIDIA 显卡时，在应用 **设置 → 环境** 下载 CUDA 加速组件（失败不影响 CPU 转写）。

### macOS 安装步骤

1. 下载 `.dmg`，双击打开。
2. 将应用拖入 **「应用程序」**。
3. 首次打开：若提示无法验证开发者 → Finder 中 **Control-点击应用 → 打开 → 仍要打开**。
4. 首次启动后按向导完成模型 seed；在「设置 → 本机 ASR / 环境」确认状态可用。

### 报错处理

#### mac：「已损坏，无法打开」/「应移到废纸篓」

多为隔离属性（quarantine），不是文件真坏了。在 **终端** 执行（按实际应用名二选一）：

```bash
sudo xattr -cr "/Applications/如是我闻.app"
```

或：

```bash
sudo xattr -cr "/Applications/rushi-desktop.app"
```

输入开机密码后回车，再从「应用程序」启动。

检查是否清掉：

```bash
xattr -l "/Applications/如是我闻.app"
```

不应再看到 `com.apple.quarantine`。

若应用还在下载文件夹，先对 `.app` 执行 `xattr -cr "路径/xxx.app"`，再拖进「应用程序」。

#### mac：「无法验证开发者」

- 优先：Control-点击 → 打开。
- 仍不行：用上面的 `xattr -cr`。
- 系统设置 → 隐私与安全性 → 若出现「仍要打开」，点一次。

#### Windows：SmartScreen「已阻止…未知应用」

「更多信息」→「仍要运行」。

#### Windows：安装包立刻失败 / 提示缺模型、要完整解压

| 错误做法 | 正确做法 |
|---|---|
| 在 zip **预览**里直接开 exe | 先 **完整解压** 再运行 |
| 只拷贝 exe，没有 `resources` | 解压后的文件夹整体保留，exe 与 `resources` **同级** |

#### Windows：解压后找不到安装包

确认解压目录里同时有：

- `如是我闻_1.0.0_Windows_x64_安装包.exe`
- `resources\bundled-asr-models\`（含 `manifest.json`）

#### 启动后无法本机转写

1. **设置 → 本机 ASR / 环境**：看侧车与模型状态。
2. 按向导「应用并重启侧车」。
3. 仍失败：导出 **诊断包（zip）** 排查。

#### 应用内更新失败

改从 CDN 手动下载对应安装包 / DMG；OTA 验签失败时不要强行装。清单地址：`https://updates.rushi.app/latest.json`。

### 平台范围

- 目前正式支持 macOS（Apple Silicon）与 Windows x64。Linux 桌面端暂未发布。
- 随包说明与上述内容一致；官网下载页会同步最新 CDN 链接。

## 2. 认识三个核心界面

Rushi 的主流程由三个界面串起来。先记住各自职责，后面的操作会更顺。

| 界面 | 你在这里做什么 |
|---|---|
| Welcome | 查看最近项目、新建项目、打开已有项目、搜索项目 |
| Project Hub | 管理音频、文本、字幕、场次信息和批量转写 |
| Editor | 对单条音频做转写、播放、波形定位、逐段校对和导出 |

典型路径：Welcome 新建或打开项目 → Project Hub 管理文件 → Editor 处理某一条音频。

### 界面切换时要注意什么

- 离开当前文件、关闭项目或退出应用前，若有未保存改动，Rushi 会先确认。
- 转写或批量转写进行中时，直接跳转会被拦截，避免任务中断或结果丢失。
- 面包屑或底部状态会显示“未保存 / 正在保存 / 已保存”，离开前先看一眼。

## 3. 创建项目与导入素材

1. 在 Welcome 选择新建项目。
2. 填写项目名称；按需要补充讲者、录制时间、地点、主题、整理人等场次信息。
3. 导入本地音频。
4. 进入 Project Hub 后，可继续导入更多音频、文本或字幕。

项目与语段保存在本地 SQLite；音频按项目归入媒体目录，便于迁移和备份。

### 重复导入与字幕匹配

- 同一文件再次导入时，Rushi 会提示你如何处理重复项。
- 导入字幕或已有转录稿时，若文件名与音频匹配，可自动附加到对应音频。
- 若匹配到多个音频，会让你选择目标文件。
- 从其他设备带回的内容包，也可在此导入以恢复项目。

## 4. 准备转写方式

Rushi 支持两类转写来源：**本机 ASR**（默认）与**在线 STT**（可选）。先选一条主路线，再开始转写。

### 本机 ASR（推荐默认）

本机 ASR 由桌面应用拉起本地 sidecar，默认地址为 `127.0.0.1:8741`。在“环境 → 本机 ASR”可查看状态、准备模型、重启运行时或清理模型缓存。

适合这些情况：

- 音频内容敏感，不希望上传云端
- 网络不稳定，或需要断网工作
- 希望长期控制成本

若状态显示模型未就绪，先完成模型准备。Windows 上 CUDA 侧车失败时，可先用 CPU 继续工作。

### 在线 STT（可选）

适合已有厂商额度、需要特定云端识别能力，或本机性能不足时使用。

当前可接入的 Provider：

- 阿里云百炼语音识别（Fun-ASR）
- 讯飞极速录音转写大模型
- OpenAI
- AssemblyAI
- Deepgram
- 自定义兼容代理

在“环境 → 在线 STT”中启用服务、选择厂商、保存密钥并探测连接。

- 讯飞需要 AppID、APIKey、APISecret
- 其他厂商通常只需 API Key
- 在线端点须使用 HTTPS；本机代理可用 `localhost` 或 `127.0.0.1`

## 5. 使用术语表提高识别稳定性

转写前，建议先把人名、地名、机构名、课程名、佛学名词等专有词加入术语表。支持的引擎会把术语作为识别偏置提交，减少“听得懂但写不对”的情况。

### 各 Provider 如何使用术语

| Provider | 术语偏置方式 |
|---|---|
| 本机 ASR | 随转写请求提交热词 |
| OpenAI | 写入转写提示词 |
| AssemblyAI | 作为 keyterms 提交 |
| Deepgram | 作为 keywords 提交 |
| 阿里云百炼 | 同步为厂商热词表 |
| 讯飞极速 | 作为热词提交 |
| 自定义代理 | 随请求提交 |

术语表回答的是“希望识别成什么正形”。校对时若反复把同一错字改成同一正确写法，Rushi 还会形成纠错记忆，供规则纠错和智能改稿使用。

## 6. 自动转写

进入 Editor，选择本机 ASR 或已配置好的在线 STT，然后开始自动转写。完成后，语段、时间码、正文和置信信息会写入项目数据库。

在 Project Hub 也可对多个文件发起批量转写；过程中可停止任务，活动记录会显示已完成、跳过和失败数量。

### 重新转写前请确认

若当前文件已有手工修改，重新自动转写前会出现确认提示——新结果可能覆盖现有语段。建议先保存或导出一份底稿，再决定是否覆盖。

批量转写时不要直接关闭应用；先停止任务或等待完成。

## 7. 在 Editor 中校对文稿

Editor 是主工作区：波形、播放头、语段列表和文本编辑绑定在一起，方便一边听一边改。

### 选择与定位

- 点击文本语段：选中该语段，并跳到语段开头
- 再次点击已选语段：只移动光标，不重复跳转
- 点击波形中的语段：选中并定位到语段开头
- 再次点击已选波形语段：按点击位置精确跳转
- 点击空白波形：移动播放头
- 键盘上下移动：只改变选择，避免打断当前播放定位

### 波形与播放

波形区支持缩放、适配全局、适配选区、迷你地图和播放跟随。长音频可先用迷你地图定位大范围，再放大处理细节。

播放支持全局播放、语段播放和循环播放。调整速度时会尽量保持人声自然，避免简单变速导致音高明显变化。

### 语段整理动作

常见校对动作：

- 修改正文
- 拆分过长语段
- 合并相邻语段
- 冻结已确认语段
- 查找与替换重复错误
- 使用规则纠错处理稳定错词
- 在定稿模式中做终检和智能改稿

冻结会保留当前分段和正文；全局通读时跳过，正文也不可直接编辑。需要继续改时先解冻。

## 8. 保存、关闭与防误操作

编辑器会显示当前保存状态。面包屑或底部出现提示时，说明仍有未保存改动、正在保存，或已经保存。

推荐习惯：

1. 每处理完一段重要内容，先确认保存状态
2. 重新转写前，先导出或保存当前稿件
3. 批量转写时，先停止或等待完成，再关闭应用

## 9. 导入已有转录稿或字幕

若已有 TXT、SRT 或其他文本稿，可从 Project Hub 或 Editor 的导入入口加入项目。

### 常见导入用法

- 把已有字幕附加到同名音频
- 用新字幕替换当前文件的旧语段
- 导入纯文本作为项目内独立文本
- 从其他设备导入 Rushi 内容包以恢复项目

导入字幕到当前 Editor 时会走替换流程；若稿件尚未保存，会先经过关闭保护提示。

## 10. 导出与交付

日常整理可用普通导出；正式交付可走定稿模式后的 Word 交付导出。

### 普通导出

- **TXT**：继续复制、整理或发给他人
- **SRT**：字幕、剪辑和视频流程
- **DOCX**：常规文档交付

### 交付 Word

从 Editor 进入“定稿模式”，完成规则纠错、智能改稿和终检后，打开“交付导出 Word”。

| 形态 | 适用场景 |
|---|---|
| 逐字稿 | 保留时间码和逐段正文，适合审校、回听、归档 |
| 讲稿 | 按自然段整理，适合阅读和发布前整理 |
| 干净稿 | 去掉时间码和低置信标记，适合最终阅读稿 |

讲者、录制时间、地点、主题等信息可进入 Word 抬头；需要时可附加修订摘要。勾选大模型润色时，会在导出过程中处理明显错别字和错误标点，并以 Word 修订轨呈现差异。

### 内容包导出

迁移或备份时可导出当前项目或整库内容包。整库包包含多个项目和词表信息，适合换电脑、备份或交接。跨 macOS / Windows 时，优先用内容包，而不是手工搬散文件。

## 11. 偏好设置

Rushi 会记住与校对体验相关的偏好，例如：

- 波形缩放比例与高度
- 转录文本字号
- 全局播放速度
- 迷你地图开关
- 播放跟随 / 文稿播放跟随方式
- 语段列表筛选
- 主题和强调色

长时间校对时，优先调好字号、波形高度和播放速度，效率提升最明显。

## 12. 迁移、备份与诊断

### 迁移媒体目录

媒体文件保存在配置的媒体目录下。换硬盘或同步到其他位置时，可调整媒体根目录，并通过项目或内容包导出完成迁移。为便于跨系统使用，Rushi 会尽量保存相对路径。

### 导出诊断包

遇到转写失败、模型准备异常、导出失败或状态不明时，可在应用内导出诊断 zip。诊断包有助于定位版本、运行时、环境和最近错误。

| 快捷键 | 作用 |
|---|---|
| Cmd/Ctrl + , | 打开设置 |
| Cmd/Ctrl + F | 查找与替换 |
| Cmd/Ctrl + J | 与下一条语段合并 |
| Cmd/Ctrl + K | 与上一条语段合并 |
| Cmd/Ctrl + D | 在播放头位置拆分语段 |

## 14. 常见问题

### 本机 ASR 显示未就绪怎么办？

到“环境 → 本机 ASR”查看模型、FFmpeg 和 sidecar 状态。模型未完成时等待复制或下载；sidecar 异常时尝试重启运行时。Windows 上 CUDA 失败可改用 CPU。

### 在线 STT 配好了但不能转写？

在“环境 → 在线 STT”探测连接。检查密钥是否已保存、网络是否可达、端点是否为 HTTPS。讯飞需确认 AppID、APIKey、APISecret 都完整。

### 转写后专名还是错？

把正确写法加入术语表，再重新转写或使用规则纠错。对已反复手改的错词，把稳定规则纳入纠错记忆。

### 文稿改乱了怎么办？

先不要重新自动转写。确认保存状态，必要时先导出 TXT 或 DOCX 留底，再决定是否替换导入、重新转写或从内容包恢复。

### 为什么不能直接离开当前页面？

若正在转写、批量转写，或存在未保存改动，Rushi 会先拦截离开操作，避免语段丢失、任务中断或新结果覆盖旧稿。

## 15. 推荐工作流

第一次使用可以按这条路线走：

1. 安装 Rushi，确认本机 ASR 就绪
2. 新建项目，补充场次信息
3. 录入常见专名到术语表
4. 导入音频并自动转写
5. 在 Editor 中按波形逐段校对
6. 对重复错词使用查找替换或规则纠错
7. 冻结已确认语段
8. 进入定稿模式做终检
9. 导出逐字稿、讲稿或干净稿
10. 需要迁移时导出项目或整库内容包

把项目、音频、语段、术语、纠错和导出都留在 Rushi 内完成，后续回看、修改和再次交付会更稳。
